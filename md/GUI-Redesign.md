# Remote-Controls GUI 现代化重设计

> 目标：基于当前 tkinter GUI 的功能与交互流程，设计一套更现代、更简洁的 UI 方案。
> 优先保留所有现有功能，仅重组布局与视觉层次，减少认知负担。

---

## 一、当前 GUI 结构分析

### 1.1 主窗口（root）

当前主窗口采用 **垂直堆叠 4 个 LabelFrame** 的布局，从上到下依次：

| 区块 | 名称 | 内容 |
|------|------|------|
| row 0 | 系统配置 | 网站(broker)、端口、认证模式(私钥/账密)、test模式、通知提示、打开任务计划、开机自启提示、自启按钮、语言选择 |
| row 1 | MQTT认证配置 | 用户名、密码、客户端ID、TLS/SSL开关、证书校验、CA证书路径、认证模式说明文字 |
| row 2 | 主题配置 | 内置主题5行(计算机/屏幕/音量/睡眠/媒体)、自定义主题Treeview、详情按钮、刷新/更多/添加/修改按钮 |
| row 3 | 操作按钮 | 打开配置文件夹、保存配置文件、取消 |

**问题总结：**
- 信息密度高、视觉层次模糊——MQTT 连接配置与认证配置被拆成两个独立框架，但用户心智模型中它们是同一件事
- 标签排列密集，右侧提示文字与左侧输入混在一起（如"需要管理员权限才能设置"和"开机自启/启用睡眠"挤在系统配置的 column 2）
- 主题配置区既包含内置主题行列表，又包含自定义 Treeview，但两者在同一行水平排列，关系不明确

### 1.2 子窗口

| 窗口 | 触发 | 内容 |
|------|------|------|
| 内置主题设置 | 点击"更多" | Computer on/off 动作 + 延时、亮度调节方案(高级亮度设置)、音量范围、睡眠 on/off 动作 + 延时 + 功能开关 |
| 高级亮度设置 | 内置主题设置内 | WMI/Dxva2/TT 模式选择、自定义顺序与策略、平滑渐进、目标、亮度范围、Twinkle Tray 配置 |
| 添加自定义主题 | 点击"添加" | 类型、状态、昵称、主题名、打开(on)/关闭(off)值、关闭预设、Hotkey设置 |
| 修改自定义主题 | 双击或点击"修改" | 同上，额外有删除按钮 |
| 详情信息 | 点击"详情" | Notebook 分三页：内置主题概览、自定义主题类型说明、使用提示 |
| 键盘录制器 | 添加/修改中点击"录制" | 按键捕获弹窗 |

## 二、现代化设计原则

1. **分组归一**：将 MQTT 连接 + 认证合并为"连接"单一区块，减少框架数量
2. **卡片化布局**：用 Tab/Notebook 或折叠卡片替代垂直堆叠，让用户按需展开
3. **视觉层级清晰**：主窗口仅展示核心连接参数 + 主题概览；所有细节设置进入子面板
4. **行动按钮就近**：保存/取消固定在底部或顶部工具栏，而非独立 row
5. **减少文字噪音**：移除右侧散落的提示性 Label，改为 hover tooltip 或副标题说明

#### 设计要点

| 改动 | 说明 |
|------|------|
| **合并 MQTT 区块** | 将"系统配置"和"MQTT认证配置"合并为单一"连接配置"卡片，认证模式切换时动态显示/隐藏用户名密码行 |
| **内置主题改为紧凑行** | 5个内置主题从逐行排列改为 2 列紧凑网格（左列：计算机+屏幕+音量，右列：睡眠+媒体），减少垂直空间占用 |
| **自定义主题改为标签/卡片列表** | Treeview 替换为可滚动的卡片/标签列表，每个项显示 `[✓] 昵称` 格式，点击展开修改弹窗 |
| **底部工具栏** | 开机自启、配置目录、保存、关闭统一放底部工具栏，贴近用户操作路径 |
| **提示文字→tooltip/副标题** | "需要管理员权限才能设置"等提示改为副标题灰色小字或 tooltip，不占独立 grid cell |
| **test/通知移入工具栏或折叠区** | test模式与通知开关不占用主视觉区，放入工具栏或连接配置底部次要行 |

## 五、交互流程变化对照

### 5.1 当前流程 → 新流程

| 操作 | 当前 | 新设计 |
|------|------|--------|
| 配置MQTT | 分别在"系统配置"和"MQTT认证配置"两个LabelFrame中填写 | 在单一"连接配置"卡片中填写，认证模式切换动态展示对应行 |
| 切换认证模式 | 下拉切换后，另一个LabelFrame中的字段禁用/启用 | 同一卡片内行动态显隐 |
| 设置TLS | 在"MQTT认证配置"中勾选，再手动点CA按钮 | TLS行内联显示，勾选后CA控件自动解锁 |
| 设置开机自启 | 系统配置区的一个独立按钮，文案动态变化 | 工具栏中的按钮，文案动态变化 |
| 设置内置主题 | 5行Checkbutton+Entry竖排，点击"更多"进子窗口 | 2列网格紧凑排列，点击"更多"进子窗口 |
| 编辑自定义主题 | Treeview列表+双击/修改按钮弹窗 | 卡片/标签列表+点击弹窗 |
| 保存配置 | 底部独立row的按钮 | 工具栏右侧按钮 |
| 看详情 | 主题配置区的"详情"按钮 | 主题配置卡片标题栏右侧按钮 |

### 5.2 子窗口调用链

```
主窗口
 ├─ 连接配置 → 认证模式切换(行显隐)
 ├─ 主题配置
 │   ├─ 内置主题 → "更多" → 内置主题设置
 │   │                      ├─ 高级亮度设置
 │   │                      ├─ 计算机控制 on/off + 延时
```

## 六、技术可行性备注

### 6.1 tkinter 限制

- tkinter 不支持真正的卡片/圆角效果，可通过 `ttk.LabelFrame` + 自定义 `ttk.Style` 模拟
- Flow/Wrap 布局的标签列表在 tkinter 中无原生组件，可用 `tk.Canvas` + 动态布局或 `ttk.Frame` + 手动网格实现
- 动态行显隐可用 `grid_remove()` / `grid()` 切换（当前代码已在认证模式切换中使用此方式）

### 6.2 建议实现策略

1. **保留 tkinter 框架**：不更换 UI 框架，仅在现有 tkinter/ttk 体系内重组
2. **自定义 ttk.Style**：统一配置颜色、字体、间距，替代零散的 `padx/pady`
3. **合并框架**：将 `system_frame` + `auth_frame` 合并为单一 `connection_frame`
4. **内置主题网格化**：从5行竖排改为2列 `grid` 排列
5. **自定义主题简化**：保留 Treeview（tkinter 中最成熟的列表方案），但增大行高 + 调整列头
6. **工具栏独立**：将底部操作按钮改为 `ttk.Frame` 工具栏，左右分区

### 6.3 可选进阶方案

| 方案 | 风格 | 代价 |
|------|------|------|
| **CustomTkinter** | 现代 Fluent 风格、圆角卡片、深色模式 | 新依赖，需大幅重写 |
| **DearPyGui** | GPU加速、高性能 | 架构完全不同 |
| **WinUI 3 (Python.NET)** | 真正 Windows 11 风格 | 极重依赖，打包复杂 |
| **Web UI (Flask + 前端)** | 完全自由设计 | 需浏览器，架构变更大 |

推荐优先在 tkinter/ttk 体系内完成重组，后续再评估是否迁移到 CustomTkinter。
