# Remote-Controls GUI 重新设计方案

## 1. 主界面布局

### 1.1 整体结构
- **顶部导航栏**：应用标题、版本信息、语言选择、帮助按钮
- **左侧侧边栏**：功能导航菜单
- **主内容区**：根据选择的功能显示相应的配置界面
- **底部状态栏**：状态信息、保存按钮、取消按钮

### 1.2 左侧导航菜单
| 菜单项 | 描述 |
|-------|------|
| 系统配置 | MQTT服务器设置、认证配置、基本设置 |
| 内置主题 | 计算机、屏幕、音量、睡眠、媒体控制 |
| 亮度调节 | 高级亮度控制设置 |
| 自定义主题 | 程序/脚本、服务、命令、热键 |
| 关于 | 版本信息、帮助文档 |

### 1.3 主内容区布局
- **系统配置**：
  - MQTT服务器设置（网站、端口）
  - 认证模式选择（私钥模式/账密模式）
  - 认证信息（用户名、密码、客户端ID）
  - TLS/SSL设置
  - 基本设置（test模式、通知提示、开机自启）

- **内置主题**：
  - 主题列表（计算机、屏幕、音量、睡眠、媒体控制）
  - 每个主题的启用状态和主题名称
  - 详细设置按钮

- **亮度调节**：
  - 控制模式选择（WMI、Dxva2、Twinkle Tray）
  - 自定义调节顺序和执行策略
  - 亮度平滑渐变设置
  - 亮度范围设置
  - Twinkle Tray配置

- **自定义主题**：
  - 主题列表（程序/脚本、服务、命令、热键）
  - 添加/修改/删除主题按钮
  - 主题详情编辑

## 2. 内置主题设置

### 2.1 计算机控制
- **打开(on)动作**：锁屏、关机、重启、注销、不执行
- **关闭(off)动作**：锁屏、关机、重启、注销、不执行
- **动作延时**：可设置执行动作前的延时时间（秒）

### 2.2 屏幕控制
- **亮度调节**：0-100%亮度调节
- **控制模式**：WMI、Dxva2、Twinkle Tray
- **高级设置**：自定义调节顺序、执行策略、平滑渐变

### 2.3 音量控制
- **音量范围**：最小音量和最大音量设置（0-100%）
- **静音控制**：支持通过pause指令快速切换静音

### 2.4 睡眠控制
- **打开(on)动作**：睡眠、休眠、关闭显示器、不执行
- **关闭(off)动作**：不执行、打开显示器、锁屏
- **动作延时**：可设置执行动作前的延时时间（秒）
- **系统睡眠功能**：启用/关闭系统睡眠/休眠功能

### 2.5 媒体控制
- **播放控制**：上一曲、下一曲、播放/暂停
- **高级控制**：支持通过百分比控制不同功能

## 3. 亮度调节设置

### 3.1 控制模式
- **WMI (系统WMI接口)**：适用于笔记本和部分支持的台式机
- **Dxva2 (物理显示器DDC/CI)**：使用物理显示器API，依赖驱动支持
- **Twinkle Tray (第三方接口)**：调用Twinkle Tray命令行控制
- **自定义组合**：可自由组合以上接口，支持同时执行或成功即止策略

### 3.2 自定义调节顺序
- **调节顺序**：可拖拽调整不同控制模式的执行顺序
- **执行策略**：同时执行（所有选中的模式）或成功即止（第一个成功的模式）

### 3.3 亮度平滑渐变
- **启用平滑渐变**：开关控制
- **步长**：1-20，控制每次调整的亮度变化量
- **间隔**：1-500ms，控制调整的时间间隔
- **分别启用**：可针对不同控制模式单独启用平滑渐变

### 3.4 亮度范围设置
- **WMI亮度范围**：最小和最大亮度值（0-100%）
- **Dxva2亮度范围**：最小和最大亮度值（0-100%）

### 3.5 Twinkle Tray配置
- **路径**：Twinkle Tray可执行文件路径
- **目标模式**：按编号、按ID、全部显示器
- **目标值**：根据目标模式设置相应的值
- **显示亮度叠加层**：开关控制

## 4. 添加/修改自定义主题

### 4.1 主题类型
- **程序或脚本**：执行EXE、.py、.ps1、.bat、.cmd等文件
- **服务(需管理员权限)**：启动/停止Windows服务
- **命令**：执行PowerShell命令
- **按键(Hotkey)**：模拟键盘按键组合

### 4.2 通用设置
- **状态**：启用/禁用主题
- **昵称**：主题的显示名称
- **主题**：主题的唯一标识

### 4.3 程序或脚本设置
- **打开(on)**：选择要执行的程序或脚本文件
- **关闭(off)**：选择关闭预设（忽略、强制结束）或自定义脚本

### 4.4 服务设置
- **服务名称**：Windows服务的名称
- **打开(on)**：启动服务
- **关闭(off)**：停止服务或自定义命令

### 4.5 命令设置
- **打开(on)**：PowerShell命令，支持{value}占位符
- **关闭(off)**：关闭预设（忽略、中断、强制结束）或自定义命令
- **窗口显示**：显示/隐藏命令窗口
- **value参数范围**：设置{value}的最小值和最大值

### 4.6 热键设置
- **打开(on)**：键盘组合，支持录制功能
- **关闭(off)**：键盘组合，支持录制功能
- **字母段间隔**：设置逐字符发送的间隔时间（ms）

## 5. 界面设计原则

### 5.1 视觉设计
- **配色方案**：使用现代、简洁的配色，主色调为蓝色系
- **字体**：使用无衬线字体，确保清晰易读
- **图标**：使用简洁的线性图标，保持风格一致
- **布局**：采用卡片式布局，合理使用留白，提高可读性

### 5.2 交互设计
- **响应式布局**：适配不同屏幕尺寸
- **实时反馈**：操作后立即显示结果
- **错误处理**：提供清晰的错误提示
- **帮助信息**：关键设置项提供工具提示
- **操作确认**：重要操作需要用户确认

### 5.3 功能设计
- **模块化**：功能分组清晰，便于导航
- **可扩展性**：预留未来功能的扩展空间
- **易用性**：简化复杂设置，提供默认值
- **一致性**：保持操作逻辑和界面风格的一致性

## 6. 技术实现建议

### 6.1 前端技术
- **框架**：使用现代前端框架（如Vue、React）
- **样式**：使用CSS预处理器（如Sass、Less）
- **响应式**：使用Flexbox或Grid布局
- **状态管理**：使用状态管理库管理应用状态

### 6.2 后端技术
- **API设计**：提供RESTful API接口
- **数据存储**：使用TOML格式存储配置
- **错误处理**：完善的错误处理机制
- **日志记录**：详细的日志记录便于调试

### 6.3 性能优化
- **懒加载**：按需加载组件和资源
- **缓存**：合理使用缓存减少重复计算
- **异步操作**：耗时操作使用异步处理
- **代码分割**：减小初始加载体积
